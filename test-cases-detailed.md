## TC-MAN001: Регистрация пользователя

| Параметр | Значение |
| --- | --- |
| Тип тестирования | Функциональное, Безопасность (Security Testing) |
| Приоритет | Высокий |
| Статус | ✅ Пройден |

### Цель теста
Проверить возможность создания нового аккаунта пользователя через веб-интерфейс приложения FatSecret.

### Предусловия

| № | Условие |
| --- | --- |
| 1 | Браузер Chrome открыт |
| 2 | Пользователь находится на главной странице https://foods.fatsecret.com |
| 3 | У пользователя есть валидный email-адрес, который ранее не использовался для регистрации |

### Шаги выполнения

| Шаг | Действие | Ожидаемый результат |
| --- | --- | --- |
| 1 | На главной странице найти и нажать кнопку "Sign In" или "Войти" | Открылась страница входа |
| 2 | На странице входа найти и нажать кнопку "Create Account" или "Создать аккаунт" | Открылась страница создания аккаунта |
| 3 | На странице создания аккаунта найти и прочитать условия Privacy Policy, согласиться с условиями (установить галочку или нажать кнопку "Accept") | Можно продолжить регистрацию |
| 4 | На странице создания аккаунта найти поле "Email", ввести валидный email-адрес (например: alisacandle_fatsecret@gmail.com), нажать кнопку "Continue" или "Продолжить" | Система приняла email и перешла к следующему шагу |
| 5 | На открывшейся странице заполнить обязательные поля анкеты профиля: Имя (First Name), Фамилия (Last Name), Дата рождения (Date of Birth), Пол (Gender), Рост (Height), Вес (Weight), Цель (Goal) - например, похудение, поддержание веса, набор веса. Нажать кнопку "Continue" или "Продолжить" | Все обязательные поля заполнены корректно |
| 6 | На финальной странице регистрации написать информацию о себе (About Me) - опционально, загрузить фото профиля (Profile Photo) - опционально, нажать кнопку "Start" или "Начать" | Регистрация завершена успешно |
| 7 | Проверить безопасность соединения: URL начинается с https://, в адресной строке отображается иконка замка, статус "Connection is secure" | Соединение защищено |

### Ожидаемый результат

| Критерий | Описание |
| --- | --- |
| Регистрация выполнена успешно | Все шаги регистрации пройдены последовательно |
| Пользователь авторизован | Пользователь авторизован и перенаправлен в личный кабинет |
| Профиль создан | Профиль создан с указанными данными |

### Фактический результат
✅ **Регистрация прошла успешно**

| Проверка | Результат |
| --- | --- |
| HTTPS — защищённое соединение | ✅ URL начинается с https:// |

### Примечания
- Для тестирования можно использовать временный email-адрес (10minutemail.com, TempMail.org)
- ⚠️ **Важно:** Использовать email с подчеркиванием "_" вместо плюса "+" (например: alisacandle_fatsecret@gmail.com)
- После теста можно удалить тестовый аккаунт

## TC-MAN002: Авторизация пользователя

| Параметр | Значение |
| --- | --- |
| Тип тестирования | Функциональное |
| Приоритет | Высокий |
| Статус | ✅ Пройден |

### Цель теста
Проверить корректность процесса входа в систему с валидными учетными данными.

### Предусловия

| № | Условие |
| --- | --- |
| 1 | Браузер Chrome открыт |
| 2 | Пользователь находится на главной странице https://foods.fatsecret.com |
| 3 | У пользователя есть существующий аккаунт с известными email и паролем |

### Шаги выполнения

| Шаг | Действие | Ожидаемый результат |
| --- | --- | --- |
| 1 | На главной странице найти и нажать кнопку "Sign In" или "Войти" | Открылась форма входа |
| 2 | В поле "Email" ввести **точно тот же email**, который был использован при регистрации. В поле "Password" ввести корректный пароль. Нажать кнопку "Sign In" или "Войти" | Данные введены корректно |
| 3 | Проверить безопасность соединения: посмотреть на адресную строку браузера - URL должен начинаться с https:// | Соединение защищено протоколом HTTPS |
| 4 | Убедиться, что пользователь успешно авторизован, проверить, что произошло перенаправление на страницу дневника или главную страницу пользователя | Пользователь авторизован и перенаправлен в личный кабинет |

### Ожидаемый результат

| Критерий | Описание |
| --- | --- |
| Авторизация выполнена успешно | Пользователь успешно вошёл в систему |
| Соединение защищено | Соединение защищено протоколом HTTPS |
| Перенаправление | Пользователь перенаправлен в личный кабинет |

### Фактический результат
✅ **Успешная авторизация, HTTPS защищено**

### Примечания
- Не рекомендуется использовать реальные пароли в тестовой документации
- ⚠️ **Важно:** Email должен точно совпадать с email, использованным при регистрации
- Для тестирования негативных сценариев можно создать отдельные тест-кейсы
- Если система показывает ошибку "неправильный логин", проверить:
  - Правильность написания email (без опечаток)
  - Использование того же email, что при регистрации
  - Отсутствие символа "+" в email (если регистрация была без него)

## TC-MAN003: Добавление продукта в дневник

| Параметр | Значение |
| --- | --- |
| Тип тестирования | Функциональное |
| Приоритет | Высокий |
| Статус | ✅ Пройден |

### Цель теста
Проверить функциональность добавления продукта в дневник питания и корректность пересчета калорий.

### Предусловия

| № | Условие |
| --- | --- |
| 1 | Пользователь авторизован в системе |
| 2 | Пользователь находится на странице дневника питания https://foods.fatsecret.com/Default.aspx?pa=m |
| 3 | Дневник открыт для текущей даты |

### Шаги выполнения

| Шаг | Действие | Ожидаемый результат |
| --- | --- | --- |
| 1 | На странице дневника нажать на "Food". Найти кнопку "Add Food" или "Добавить продукт" - соответственно в разделах - Завтрак, Обед, Ужин или Другое. Нажать на кнопку | Открылось окно или форма поиска продуктов |
| 2 | В поле поиска ввести название продукта (например: "яблоко" или "apple"). Нажать кнопку поиска | Появился выпадающий список результатов поиска |
| 3 | Просмотреть список найденных продуктов. Выбрать нужный продукт из списка (например: "Gala Apples"). Нажать на выбранный продукт и выбрать количество или вес продукта. Далее - "Add selected" или "Добавить" | Продукт выбран и добавлен |
| 4 | Убедиться, что продукт появился в дневнике питания. Проверить, что отображается правильное количество продукта. Проверить, что отображаются калории и макронутриенты (белки, жиры, углеводы) | Продукт отображается корректно |
| 5 | Проверить общее количество калорий за день в Day Summary внизу страницы. Убедиться, что калории добавленного продукта учтены в общей сумме. Проверить, что макронутриенты также пересчитаны | Калории и макронутриенты пересчитаны |

### Ожидаемый результат

| Критерий | Описание |
| --- | --- |
| Продукт успешно добавлен в дневник | Продукт отображается в списке |
| Количество продукта отображается корректно | Указанное количество сохранено |
| Калории и макронутриенты пересчитаны | Значения отображаются правильно |
| Общая сумма калорий за день обновлена | Day Summary показывает актуальные данные |

### Фактический результат
✅ **Продукт добавлен, калории пересчитаны корректно**

### Примечания
- Рекомендуется тестировать с разными продуктами и единицами измерения
- Можно проверить добавление продукта в разные приемы пищи (завтрак, обед, ужин)

## TC-MAN005: Отслеживание веса

| Параметр | Значение |
| --- | --- |
| Тип тестирования | Функциональное |
| Приоритет | Средний |
| Статус | ✅ Пройден |

### Цель теста
Проверить функциональность добавления записи о весе и обновления графика изменения веса.

### Предусловия

| № | Условие |
| --- | --- |
| 1 | Пользователь авторизован в системе |
| 2 | Пользователь находится в разделе отслеживания веса |
| 3 | У пользователя есть доступ к функции отслеживания веса |

### Шаги выполнения

| Шаг | Действие | Ожидаемый результат |
| --- | --- | --- |
| 1 | В главном меню найти раздел "Weight" или "Вес". Нажать на раздел | Открылась страница отслеживания веса |
| 2 | Проверить наличие графика изменения веса. Зафиксировать текущее состояние графика (если есть предыдущие записи) | График отображается |
| 3 | Найти кнопку "Add Weight" или "Добавить вес". Нажать на кнопку. В открывшейся форме ввести текущий вес (например: 70.5 кг). Указать дату измерения (по умолчанию должна быть текущая дата). Нажать кнопку "Save" или "Сохранить" | Форма открылась, данные введены |
| 4 | Убедиться, что запись о весе добавлена в список. Проверить, что вес отображается корректно. Проверить, что дата измерения указана правильно | Запись сохранена корректно |
| 5 | Проверить, что график обновился с новой точкой. Убедиться, что новая запись отображается на графике. Проверить, что график корректно показывает изменение веса во времени | График обновлён |

### Ожидаемый результат

| Критерий | Описание |
| --- | --- |
| Запись о весе успешно добавлена | Запись отображается в списке записей |
| График обновлен | График обновлен с новой точкой данных |
| График корректно отображает изменение веса | Данные визуализированы правильно |

### Фактический результат
✅ **Запись добавлена, график обновлён корректно**

### Примечания
- Рекомендуется проверить редактирование и удаление записей о весе
- Можно проверить экспорт данных о весе (если доступно)

## TC-MAN007: Редактирование записей в дневнике

| --- | --- |
| Тип тестирования | Функциональное |
| Приоритет | Средний |
| Статус | ✅ Пройден |

Проверить функциональность редактирования записей в дневнике питания и корректность пересчета калорий после изменения.

### Предусловия

| № | Условие |
| --- | --- |
| 1 | Пользователь авторизован в системе |
| 2 | Пользователь находится на странице дневника питания |
| 3 | В дневнике есть хотя бы одна запись о продукте (можно добавить через TC-MAN003) |

### Шаги выполнения

| Шаг | Действие | Ожидаемый результат |
| --- | --- | --- |
| 1 | Перейти на страницу дневника питания. Убедиться, что дневник открыт для текущей даты. Проверить наличие записей о продуктах | Дневник открыт, записи присутствуют |

| 3 | Найти кнопку "Edit" или "Редактировать" рядом с выбранной записью. Или нажать на саму запись (если она кликабельна) | Открылась форма редактирования |
| 4 | В форме редактирования изменить количество продукта (например: было 100г, изменить на 150г). Или изменить единицу измерения (если доступно). Проверить, что изменения отображаются в форме | Изменения отображаются |

| 7 | Проверить общее количество калорий за день. Убедиться, что калории пересчитаны с учетом нового количества продукта | Общие калории пересчитаны |
| 8 | Перезагрузить страницу (F5). Убедиться, что изменения сохранены. Проверить, что отредактированная запись отображается с новыми значениями | Изменения сохранены после перезагрузки |

### Ожидаемый результат

| Критерий | Описание |
| --- | --- |
| Запись успешно отредактирована | Новое количество отображается |
| Калории и макронутриенты пересчитаны правильно | Значения соответствуют изменениям |
| Общая сумма калорий за день обновлена | Day Summary актуален |
