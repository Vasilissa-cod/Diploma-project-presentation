# Детальное описание автотестов

## Тест-кейс 1 — Проверка ошибки при пустых полях (негативный)

**Назначение:** проверка валидации формы авторизации при попытке входа с пустыми полями логина и пароля.

**Ожидаемый результат:** система отображает alert с сообщением об ошибке, указывающим на необходимость заполнения обязательных полей.

**Шаги:**
1. Перейти на страницу авторизации (клик по "Sign In")
2. Очистить поля логина и пароля
3. Нажать кнопку входа
4. Проверить появление alert с текстом ошибки

## Тест-кейс 3 — Поиск продуктов (позитивный)

**Назначение:** проверка функциональности поиска продуктов по названию.

**Ожидаемый результат:** на странице результатов поиска отображаются продукты, в названии которых содержится указанная подстрока.

**Ресурсы:**
- Поисковый запрос: "tomato"

**Шаги:**
1. Открыть страницу поиска продуктов
2. Очистить поле поиска и ввести "tomato"
3. Нажать кнопку поиска
4. Проверить, что в результатах присутствует текст "tomato"

## Тест-кейс 4 — Подсчёт калорий (позитивный)

**Назначение:** проверка функциональности добавления продукта в дневник питания и отображения калорий.

**Ожидаемый результат:** после добавления продукта на странице дневника отображается информация о калориях.

**Ресурсы:**
- Продукт: strawberries

**Шаги:**
1. Авторизоваться в системе
2. Открыть страницу дневника питания
3. Добавить продукт "strawberries" в завтрак
4. Проверить, что на странице отображается информация о калориях

---

## Тест-кейс 5 — Добавление записи в журнал (позитивный)

**Назначение:** проверка функциональности добавления текстовой записи в личный журнал пользователя.

**Ожидаемый результат:** добавленная запись сохраняется и отображается на странице журнала.

**Ресурсы:**
- Текст записи: "New record from test"

**Шаги:**
1. Авторизоваться в системе
2. Открыть страницу создания новой записи в журнале
3. Ввести текст записи
4. Сохранить запись
5. Проверить, что запись отображается на странице журнала
