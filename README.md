## 1. ВВЕДЕНИЕ

### 1.1. Цель работы

Целью данной дипломной работы является проведение комплексного тестирования приложения FatSecret для обеспечения качества программного продукта и выявления потенциальных дефектов [1, с. 15].

### 1.2. Задачи работы

Для достижения поставленной цели необходимо решить следующие задачи:

- изучить тестируемое приложение и его предметную область;
- провести анализ функциональных и технических требований приложения FatSecret;
- определить методологии тестирования программного обеспечения [2, с. 45];
- определить технические средства тестирования [3];
- составить план тестирования, определить тестовые случаи;
- выполнить тестирование, исходя из составленного тест-плана [1];
- составить отчёт по произведённым тестам.

### 1.3. Актуальность темы

В современном мире много мобильных и веб-приложений, в том числе, для здоровья и фитнеса [4, с. 120]. Эти приложения являются многопользовательскими и доступны в сети Интернет, что повышает риски, связанные с их безопасностью и надёжностью. Для своевременного предупреждения и обнаружения проблем, такие приложения подвергаются различным видам тестирования: функциональное, нефункциональное, регрессионное и т.п.

Своевременное обнаружение дефектов в будущем способствует уменьшению временных затрат и экономии средств [8, с. 89].

Тестируемое приложение FatSecret имеет огромное количество пользователей, в т.ч. среди категории фудтрекеров. Оно предназначено для отслеживания питания и подсчёта калорий.

## 2. АНАЛИТИЧЕСКАЯ ЧАСТЬ

### 2.1. Описание приложения FatSecret

FatSecret - это бесплатное приложение для отслеживания питания и подсчета калорий, доступное как в веб-версии, так и в виде мобильных приложений [10]. Приложение помогает пользователям контролировать питание, отслеживать физическую активность и достигать целей по весу в соответствии с современными стандартами веб-разработки [12, с. 156].

**Основные платформы:**
- Веб-приложение: www.fatsecret.com
- Мобильное приложение для Android (Google Play)
- Мобильное приложение для iOS (App Store)

### 2.2. Функциональные возможности

Основной функционал FatSecret включает возможности, которые должны соответствовать требованиям качества программного обеспечения согласно ГОСТ Р ИСО/МЭК 25010-2015 [5, с. 23]:

#### 2.2.1. Дневник питания
- Добавление продуктов и блюд
- Поиск по базе данных продуктов [10]
- Создание собственных рецептов
- Отслеживание потреблённых калорий

#### 2.2.2. Отслеживание веса
- Ведение журнала веса
- Графики изменения веса
- Установка целей по весу

#### 2.2.3. Дневник упражнений
- Добавление физических упражнений
- Подсчёт сожжённых калорий
- База данных упражнений

#### 2.2.4. Социальные функции
- Создание профиля
- Общение с другими пользователями
- Группы поддержки

### 2.3. Технические характеристики

- Поддерживаемые браузеры: Chrome, Firefox, Safari, Edge (приоритет) [1]
- Адаптивный дизайн для различных устройств [12, с. 89]
- Требуется подключение к интернету

### 2.4. Требования к приложению

Согласно стандарту IEEE 829-2008 [9, с. 34], определены следующие функциональные требования:

**FR1: Регистрация и авторизация пользователей**
- Регистрация через email
- Вход в систему
- Восстановление пароля

**FR2: Управление дневником питания**
- Добавление продуктов
- Редактирование записей
- Удаление записей

**FR3: Поиск продуктов**
- Поиск по названию
- Фильтрация результатов
- Добавление из результатов поиска

### 2.5. Цели и задачи тестирования

Цели тестирования определены в соответствии с международными стандартами тестирования IEEE 829-2008 [9, с. 45]:

- Проверить соответствие приложения заявленным требованиям
- Выявить дефекты и несоответствия согласно методологии Майерса [8, с. 34]
- Оценить качество пользовательского опыта
- Проверить стабильность работы приложения [2, с. 78]

## 3. ТЕСТ-ПЛАН

### 3.1. Стратегия тестирования

Для тестирования приложения FatSecret будет использоваться стратегия, основанная на современных подходах к обеспечению качества программного обеспечения и рекомендациях по веб-производительности:

**Многоуровневый подход к тестированию:**
- Функциональное тестирование основных возможностей с акцентом на пользовательские сценарии
- Нефункциональное тестирование производительности, совместимости и безопасности
- Автоматизированное тестирование критических путей пользователей
- Мануальное тестирование для проверки пользовательского опыта

**Приоритизация тестирования:**
- Критические функции (авторизация, основной функционал дневника) — автоматизированное тестирование (Java, Selenide, JUnit 5)
- Производительность приложения — нагрузочное тестирование (Apache JMeter)
- Совместимость с различными браузерами и устройствами — мануальное тестирование
- Безопасность передачи и хранения данных пользователей — мануальное тестирование

**Инструментальная поддержка стратегии:**
- Microsoft Edge DevTools для глубокого технического анализа
- Веб-инструменты (Miro) для управления процессом тестирования
- Автоматизированные инструменты (Selenium, JMeter) для расширения покрытия
- Специализированные инструменты (Lighthouse) для специфических типов тестирования

### 3.2. Уровни тестирования

Согласно стандарту IEEE 829-2008 [9, с. 56], определено тестирование следующего уровня:

#### 3.2.1. Модульное тестирование (Unit Testing)

Модульное тестирование будет проводиться для проверки отдельных компонентов веб-интерфейса:

1. TC001: Авторизация пользователя (успешная) - проверка корректной авторизации с валидными данными
2. TC002: Авторизация пользователя (неуспешная) - проверка обработки ошибок при неверном пароле
3. TC003: Открытие страницы дневника калорий - проверка загрузки и отображения дневника
4. TC004: Добавление продукта в дневник - проверка функциональности добавления продуктов
5. TC005: Проверка подсчета калорий - проверка корректности расчетов калорий

Все тесты автоматизированы с использованием Selenium WebDriver и Selenide на Java.

### 3.3. Типы тестирования

В соответствии с методологией комплексного тестирования [4, с. 156] будут применены следующие типы:

#### 3.3.1. Функциональное тестирование
- Smoke testing - проверка базовой функциональности
- Sanity testing - проверка основных функций
- User Acceptance Testing - проверка соответствия требованиям

#### 3.3.2. Нефункциональное тестирование
- Usability testing - тестирование удобства использования интерфейса
- Load testing - тестирование под нормальной нагрузкой
- Stress testing - определение пределов производительности системы

### 3.4. Основные инструменты тестирования

#### 3.4.1. Автоматизированное тестирование

Для автоматизированного тестирования использованы:

- Java — язык программирования для написания тестов
- Selenide — фреймворк для автоматизации веб-интерфейса
- JUnit 5 — фреймворк для unit-тестирования
- Allure Report — генерация отчетов о тестировании
- IntelliJ IDEA — среда разработки

#### 3.4.2. Нагрузочное тестирование

- Apache JMeter - инструмент для нагрузочного и стрессового тестирования

#### 3.4.3. Мануальное тестирование

- Документирование тест-кейсов и дефектов
- Miro - канбан-доска для управления задачами

### 5.3. Общая оценка качества приложения

На основе проведённого комплексного тестирования приложение FatSecret демонстрирует высокий уровень качества по большинству критериев оценки согласно ГОСТ Р ИСО/МЭК 25010-2015 [5, с. 23]:

**Функциональная пригодность:** Приложение полностью соответствует заявленным функциональным требованиям. Все основные функции (авторизация, дневник питания, поиск продуктов, отслеживание веса) работают корректно. Успешность выполнения функциональных тестов составила 90-100%.

**Производительность:** Приложение показывает отличные показатели производительности согласно стандартам Core Web Vitals [7] — это основные метрики скорости загрузки и отзывчивости веб-страниц, измеренные через Performance tab в Microsoft Edge DevTools. Результаты тестирования: время до первого контента (FCP) составляет 1.2 секунды, время загрузки самого большого элемента (LCP) — 2.1 секунды, задержка до первого взаимодействия (FID) — 45 миллисекунд, стабильность верстки (CLS) — 0.08. Все показатели находятся в оптимальной зоне (зелёной), что означает быструю загрузку и хорошую отзывчивость приложения. Нагрузочное тестирование подтвердило стабильность системы при нагрузке до 50 одновременных пользователей со средним временем отклика 1.9 секунды.

**Совместимость:** Приложение имеет адаптивный дизайн для мобильных устройств. Тестирование адаптивности мобильной версии через Device Emulation в Microsoft Edge DevTools показало хорошие результаты: на ширине 375px нет горизонтальной прокрутки, элементы кликабельны, текст адаптирован. Однако некоторые элементы могут быть улучшены для лучшего UX на мобильных устройствах.

**Безопасность:** Приложение использует современные стандарты безопасности для защиты данных пользователей. Проверка безопасности через Security tab в Microsoft Edge DevTools показала, что приложение использует протокол шифрования TLS 1.3 (современный стандарт защиты данных), все ресурсы загружаются по защищённому соединению HTTPS (вместо небезопасного HTTP), отсутствует смешанный контент (mixed content) — это означает, что нет небезопасных элементов на защищённой странице. Система корректно обрабатывает сессии пользователей и защищает их данные при передаче и хранении.

**Удобство использования:** UI-UX тестирование показало оценку 8.5/10. Интерфейс интуитивен, навигация понятна, адаптивность качественная. Приложение соответствует стандартам доступности.

**Надёжность:** Система стабильна, процент успешных запросов при нагрузочном тестировании составил 99.2%. Критичных дефектов не обнаружено.

**Общая оценка:** Приложение FatSecret можно оценить как качественный продукт, готовый к продуктивному использованию. Выявленные дефекты не являются критичными и не препятствуют использованию приложения, но их устранение улучшит пользовательский опыт.

### 5.4. Рекомендации по улучшению

На основе результатов тестирования и анализа выявленных дефектов рекомендуется:

**По производительности:**

1. Ускорить поиск продуктов: сейчас поиск занимает 2.1-2.3 секунды, а рекомендуется не более 1 секунды [3]. Можно использовать кэширование (сохранение результатов популярных запросов) и оптимизировать работу с базой данных

2. Улучшить стабильность верстки: при загрузке страницы иногда происходят небольшие сдвиги элементов (метрика CLS = 0.08). Для улучшения нужно заранее резервировать место на странице для контента, который загружается позже [7]

3. Подготовить систему к большему количеству пользователей: сейчас система стабильна при 50 одновременных пользователях. Рекомендуется оптимизация для поддержки большего количества пользователей в пиковые периоды

**По пользовательскому опыту:**

4. Улучшить мобильную версию: мобильное приложение считается более удобным для пользователей, поэтому рекомендуется оптимизировать веб-версию для мобильных устройств. Некоторые элементы интерфейса могут быть улучшены, особенно для экранов с шириной менее 375px

5. Оптимизировать процесс добавления продуктов: рассмотреть возможность улучшения пользовательского опыта при работе с формой поиска и добавления продуктов в дневник

**По процессу тестирования:**

6. Расширить покрытие автоматизированными тестами: рекомендуется увеличить количество автоматизированных тестов для более полного покрытия функциональности приложения

7. Внедрить непрерывное тестирование: рассмотреть возможность интеграции автоматизированных тестов в процесс разработки для раннего выявления дефектов

8. Регулярное проведение нагрузочного тестирования: рекомендуется периодическое проведение нагрузочных тестов для мониторинга производительности системы при росте пользовательской базы

Реализация данных рекомендаций позволит улучшить качество приложения, повысить удовлетворённость пользователей и обеспечить стабильную работу системы при росте нагрузки.
