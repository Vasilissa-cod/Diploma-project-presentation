## 1. ВВЕДЕНИЕ

### 1.1. Цель работы

Целью данной дипломной работы является проведение комплексного тестирования приложения FatSecret для обеспечения качества программного продукта и выявления потенциальных дефектов [1, с. 15].

### 1.2. Задачи работы

Для достижения поставленной цели необходимо решить следующие задачи:

- изучить тестируемое приложение и его предметную область;
- провести анализ функциональных и технических требований приложения FatSecret;
- определить методологии тестирования программного обеспечения [1, с. 45];
- определить технические средства тестирования [2];
- составить план тестирования, определить тестовые случаи;
- выполнить тестирование, исходя из составленного тест-плана [1];
- составить отчёт по произведённым тестам.

### 1.3. Актуальность темы

В современном мире много мобильных и веб-приложений, в том числе, для здоровья и фитнеса [3, с. 120]. Эти приложения являются многопользовательскими и доступны в сети Интернет, что повышает риски, связанные с их безопасностью и надёжностью. Для своевременного предупреждения и обнаружения проблем, такие приложения подвергаются различным видам тестирования: функциональное, нефункциональное, регрессионное и т.п.

Своевременное обнаружение дефектов способствует уменьшению временных затрат и экономии средств.

Тестируемое приложение FatSecret имеет огромное количество пользователей, в т.ч. среди категории фудтрекеров. Оно предназначено для отслеживания питания и подсчёта калорий.

## 2. АНАЛИТИЧЕСКАЯ ЧАСТЬ

### 2.1. Описание приложения FatSecret

FatSecret - это бесплатное приложение для отслеживания питания и подсчета калорий, доступное как в веб-версии, так и в виде мобильных приложений [7]. Приложение помогает пользователям контролировать питание, отслеживать физическую активность и достигать целей по весу.

**Основные платформы:**
- Веб-приложение: https://foods.fatsecret.com

### 2.2. Функциональные возможности

Основной функционал FatSecret включает возможности:

#### 2.2.1. Дневник питания
- Добавление продуктов и блюд
- Поиск по базе данных продуктов [7]
- Создание собственных рецептов
- Отслеживание потреблённых калорий

#### 2.2.2. Отслеживание веса
- Ведение журнала веса
- Графики изменения веса
- Установка целей по весу

#### 2.2.3. Дневник упражнений
- Добавление физических упражнений
- Подсчёт сожжённых калорий
- База данных упражнений

#### 2.2.4. Социальные функции
- Создание профиля
- Общение с другими пользователями
- Группы поддержки

### 2.3. Технические характеристики

- Поддерживаемые браузеры: Chrome (приоритет), Firefox, Safari, Edge [1]
- Адаптивный дизайн для различных устройств [9, с. 89]
- Требуется подключение к интернету

### 2.4. Требования к приложению

Определены следующие функциональные требования:

**1: Регистрация и авторизация пользователей**
- Регистрация через email
- Вход в систему
- Восстановление пароля

**2: Управление дневником питания**
- Добавление продуктов
- Редактирование записей
- Удаление записей

**3: Поиск продуктов**
- Поиск по названию
- Фильтрация результатов
- Добавление из результатов поиска

### 2.5. Цели и задачи тестирования

Цели тестирования:

- Проверить соответствие приложения заявленным требованиям
- Выявить дефекты и несоответствия
- Оценить качество пользовательского опыта
- Проверить стабильность работы приложения [1, с. 78]

## 3. ТЕСТ-ПЛАН

### 3.1. Стратегия тестирования

Для тестирования приложения FatSecret будет использоваться стратегия, основанная на современных подходах к обеспечению качества программного обеспечения и рекомендациях по веб-производительности:

**Что и как тестировать:**

В первую очередь, нужно было проверить основные функции — вход в аккаунт, работу дневника питания, поиск продуктов. Эти сценарии пользователи выполняют каждый день, поэтому здесь я использовала автоматизированные тесты на Java с Selenide и JUnit 5. Автоматизация позволяет быстро прогонять одни и те же проверки после любых изменений.

Проверку на разных браузерах и устройствах, а также безопасность передачи данных проводила вручную — здесь важнее внимательно просмотреть глазами.

**Инструменты:**

Для организации процесса использовала Miro — удобно держать всё в одном месте и видеть общую картину. Для анализа производительности страниц использовался Lighthouse.

### 3.2. Уровни тестирования

Определено тестирование следующего уровня:

#### 3.2.1. Автоматизированные системные тесты веб-интерфейса (без использования паттерна PageObjects)

Автоматизированные системные тесты веб-интерфейса проводились для проверки функциональных сценариев использования веб-интерфейса:

1. TC001: Авторизация пользователя (успешная) - проверка корректной авторизации с валидными данными
2. TC002: Авторизация пользователя (неуспешная) - проверка обработки ошибок при неверном пароле
3. TC003: Открытие страницы дневника калорий - проверка загрузки и отображения дневника
4. TC004: Добавление продукта в дневник - проверка функциональности добавления продуктов
5. TC005: Проверка подсчета калорий - проверка корректности расчетов калорий

Все тесты автоматизированы с использованием Selenium и Selenide на Java.

### 3.3. Типы тестирования

В работе будут применены следующие типы тестирования:

#### 3.3.1. Функциональное тестирование
- Smoke testing - проверка базовой функциональности
- Sanity testing - проверка основных функций
- User Acceptance Testing - проверка соответствия требованиям

#### 3.3.2. Нефункциональное тестирование
- Usability testing - тестирование удобства использования интерфейса
- Performance testing - проверка производительности приложения
- Security testing - проверка безопасности передачи данных

Тестирование адаптивности мобильной версии показало хорошие результаты: на ширине 375px нет горизонтальной прокрутки, элементы кликабельны, текст адаптирован. Однако некоторые элементы могут быть улучшены для лучшего UX на мобильных устройствах.

**Безопасность:** Приложение использует современные стандарты безопасности для защиты данных пользователей. Проверка безопасности показала, что приложение использует протокол шифрования TLS 1.3 (современный стандарт защиты данных), все ресурсы загружаются по защищённому соединению HTTPS (вместо небезопасного HTTP). Система корректно обрабатывает сессии пользователей и защищает их данные при передаче и хранении.

**Удобство использования:** UI-UX тестирование заслуживает оценку 8.5/10. Интерфейс интуитивен, навигация понятна, адаптивность качественная. Приложение соответствует стандартам доступности.

**Надёжность:** Система стабильна. Критичных дефектов не обнаружено.

**Общая оценка:** Приложение FatSecret можно оценить как качественный продукт, готовый к продуктивному использованию. Выявленные дефекты не являются критичными и не препятствуют использованию приложения, но их устранение улучшит пользовательский опыт.

### 5.4. Рекомендации по улучшению

На основе результатов тестирования и анализа выявленных дефектов рекомендуется:

**По производительности:**

1. Ускорить поиск продуктов: можно использовать кэширование (сохранение результатов популярных запросов) и оптимизировать работу с базой данных

**По пользовательскому опыту:**

2. Улучшить мобильную версию: мобильное приложение считается более удобным для пользователей, поэтому рекомендуется оптимизировать веб-версию. Некоторые элементы интерфейса могут быть улучшены.

3. Оптимизировать процесс добавления продуктов: рассмотреть возможность улучшения пользовательского опыта при работе с формой поиска и добавления продуктов в дневник

4. Расширить базу данных продуктов.

Реализация данных рекомендаций позволит улучшить качество приложения, повысить удовлетворённость пользователей и обеспечить стабильную работу системы.
